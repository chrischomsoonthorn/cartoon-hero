Learning 6 - Static, Relative and Absolute Positioning

This is probably going to be a big post... Positioning in CSS and HTML was a little bit tricky to get my head around. The attribute of positioning i.e. positioning: relative; for instance differs a lot depending on which one you select.

Below is an example of STATIC positioning:

This is the parent relative box
This is static child 1
This is static child 2
This is static child 3

Above we can see that, along with the necessary margining and padding added to the elements, all of the static elements take up 100% of the width of the object and resize themselves appropriately to fit the screen size which is shown, which is pretty awesome as we don't need to go and manually adjust the height and width and so on. It is 'always positioned with normal accordance to the page.'

Below is an example of RELATIVE positioning:

This is the parent relative box
This is relative child 1 - moved from the left by 10px
This is relative child 2 - moved from the left by 20px - you can see how using relative positioning allows us to change left/right/top/bottom values
This is relative child 3 - it's left value exceeds that of the boundaries of the parent static box

Relative positioning allows the user to adjust the element's top/bottom/right/left values. Importantly this does not affect how other elements 'perceive' the relative element to move. This is why it goes out of the parent box. The parent box still 'thinks' the element is in it's normal place.

Below is an example of ABSOLUTE positioning:

This is the parent relative box
This is absolute child 1
This is absolute child 2
This is absolute child 3
This is absolute child 4

Similar to floating an element, absolute positioning allows the user to specifically customise exactly where an element will go. The box that the absolute element is also sized appropriately to the content inside. Absolute elements will 'cling' to the nearest relative element for reference. In this case it clings to the parent relative box. If we wanted to remove the relative attribute from the parent box we would end up with the child absolute boxes at the top and bottom of the page respectively.
